## 3. Детализация (Technical Implementation Plan)

### A. Создать серверные `layout.tsx` для каждой услуги
Сейчас структура: `app/usluga/page.tsx` ("use client").
**Решение (Risk: Low):**
Создать файлы `app/razval-shozhdenie/layout.tsx` (Server Component), который:
1.  Экспортирует `metadata` (Title, Desc).
2.  Инжектит JSON-LD Schema (`Service` + `FAQPage`).
3.  Оборачивает `{children}` (сам `page.tsx` с UI).

**Пример JSON-LD:**
```javascript
{
  "@context": "https://schema.org",
  "@type": "Service",
  "name": "3D Развал-схождение",
  "provider": { "@type": "AutoRepair", "name": "Avto14" },
  "areaServed": "Казань",
  "description": "...",
  "hasOfferCatalog": {
     "@type": "OfferCatalog",
     "name": "Услуги",
     "itemListElement": [ ...FAQ as CreativeWork... ] // Или отдельно FAQPage
  }
}
```

### B. FAQPage Schema
Поскольку FAQ контент лежит внутри компонента `page.tsx` (prop), его нужно задублировать в `layout.tsx` для SEO. Это безопасно (Google разрешает дублировать данные, если они соответствуют контенту на странице).
